HTML

  • HTML is used to create the structure of a webpage
  • HTML is made up of elements which work together to provide content and the basic framework for all pages on the web
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The elements seen on a page include text, image, list and headers
  • The elements used to divide and describe a page are meta-related, content division and semantic
  • lang-en tells what type of language will be used for webpage
  • lang-en is an html attribute
  • Attributes define additional info about an element
  • HTML attributes are always defined in the beginning element
  • Metadata is data about the webpage that the browser uses to effectively serve our webpage to users
  • Metadata is not seen by users and contains important info about accessibility, search engines and performance

Git

  • Git is a version control system that let's you manage and keep track of each version of your project
  • Cit commands always start with "git"
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • GitHub is a cloud-based hosting service that allows you to keep track of projects, store code and collaborate with others
  • repository: any project in GitHub
  • branch: wworkspace where you can make changes that don't affect the live site
  • markdown: the way content files are formatted in GitHub
  • commit changes: saving a new version of a file
  • pull request: asking for changes on a branch to merge into another branch
  • merge: pull request is approved and pulled into main branch
  • issue: how work is tracked and a way to report new tasks or content fixes
